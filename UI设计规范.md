# 手冲咖啡APP —— UI设计规范

> 现代暖调精致 × 咖啡色系 × 无衬线 × Material细线图标

---

## 一、设计理念

**一句话**：像一家精品咖啡馆出品的App——温暖、克制、注重细节，让每一次打开都像推开咖啡馆的门。

**三个关键词**：
- **暖**：咖啡色系传递温度，不冰冷
- **精**：细线图标、微妙阴影、留白得当，精致不拥挤
- **轻**：交互轻盈，微动效润物无声

## 三、字体排印

### 3.1 字体族

```
标题  → 默认系统无衬线 (sans-serif-medium)
正文  → 默认系统无衬线 (sans-serif)
数字  → 等宽数字变体 (tabular numbers，用于计时器/克重)
```

> 不引入自定义字体，保持包体积小、加载快。系统默认无衬线在现代 Android 上效果已足够好。

### 3.2 字号阶梯（Material3 Type Scale）

| 层级 | 字号 | 字重 | 用途 |
|------|------|------|------|
| `displayLarge` | 32sp | 400 | 冲煮完成大标题 |
| `headlineMedium` | 24sp | 500 | 页面标题、方案名 |
| `titleLarge` | 20sp | 500 | 卡片主标题 |
| `titleMedium` | 16sp | 500 | 卡片副标题、步骤动作类型 |
| `bodyLarge` | 16sp | 400 | 正文、参数值 |
| `bodyMedium` | 14sp | 400 | 次要信息、标签文字 |
| `labelLarge` | 14sp | 500 | 按钮文字 |
| `labelMedium` | 12sp | 500 | 小标签、徽章、状态标签 |
| `labelSmall` | 10sp | 400 | 辅助小字（克单价等） |

### 3.3 行高与段距

- 正文行高：1.5x 字号
- 标题行高：1.2x 字号
- 卡片内间距：12dp
- 段间间距：8dp

## 四、间距与圆角系统

### 4.1 间距阶梯（8dp 基准）

```
xs  = 4dp    图标与文字间距
sm  = 8dp    标签间距、紧密元素
md  = 12dp   卡片内边距
lg  = 16dp   页面水平边距、卡片间距
xl  = 24dp   区块间距
xxl = 32dp   页面顶部/底部留白
```

### 4.2 圆角阶梯

```
sm  = 8dp    按钮、小标签、徽章
md  = 12dp   卡片、输入框、弹窗
lg  = 16dp   大卡片（引导页面中心卡片）
full = 50%   药丸形（FAB、状态标签、快捷按钮）
```

### 4.3 阴影层级

| 层级 | Elevation | 用途 |
|------|-----------|------|
| 0 | 0dp | 页面背景、列表底 |
| 1 | 2dp | 卡片（默认） |
| 2 | 4dp | 卡片（悬浮/按压） |
| 3 | 8dp | 弹窗、底部抽屉 |
| 4 | 12dp | FAB |

> 所有阴影使用咖啡色 `#5D4037` 带 8%-15% 透明度，而非纯黑阴影，保持暖调。

## 八、动效规范

### 8.1 微交互

| 场景 | 动效 | 时长 |
|------|------|------|
| 卡片按压 | 缩放到 97% + 阴影加深 | 150ms |
| 页面切换 | 水平滑动（标准 Navigation） | 300ms |
| FAB 出现 | 缩放 + 淡入 | 250ms |
| 星级点击 | 弹跳（scale 0.8→1.15→1.0） | 200ms |
| 步骤拖动 | 跟随手指 + 其他步骤让位 | 实时(spring) |
| 删除确认 | 卡片右滑淡出 | 300ms |
| 数值变化 | 数字滚动/溶解切换 | 200ms |
| 圆环进度 | 平滑过渡，非跳跃 | spring动画 |
| 按钮点击 | 涟漪（Ripple，咖啡色） | 系统默认 |

### 8.2 页面转场

- 标准进入：从右侧滑入（Navigation 默认）
- 引导开始：卡片从中心展开放大（共享元素过渡）
- 引导完成：淡入 + 轻微上移
- 弹窗/对话框：底部弹出（BottomSheet 风格）

### 8.3 动效原则

1. 不炫技——动效为功能服务
2. 不拖沓——任何动画不超过 300ms
3. 有反馈——每个可交互元素都有按压态
4. 可中断——使用 `animate*AsState` 确保可被新的目标值打断

## 九、空状态与错误状态

### 空状态（三Tab统一风格）

```
        ☕
   （大号图标，浅烘色，半透明）
   
   还没有方案
   创建你的第一个冲煮方案吧
   
      [+ 新建方案]
```

三个Tab的空状态文案：

| Tab | 主文案 | 副文案 | 按钮 |
|-----|--------|--------|------|
| 方案库 | 还没有方案 | 创建你的第一个冲煮方案吧 | + 新建方案 |
| 冲煮记录 | 还没有记录 | 冲一杯咖啡，开始记录吧 | + 新建记录 |
| 豆仓 | 还没有豆子 | 添加你的第一包咖啡豆吧 | + 添加豆子 |

### 错误状态

- 数据库操作失败：Snackbar 提示 "操作失败，请重试"，底部带重试按钮
- 必填字段为空：输入框变红 + 下方红字提示 "此项必填"
- 水温超过100℃：输入框变红 + "水温不能超过100℃"
- 数值为负：输入框变红 + "不能为负数"
- 豆子库存不足：弹窗 "XX 剩余仅 Xg，当前用豆量 Xg，仍要保存吗？"
