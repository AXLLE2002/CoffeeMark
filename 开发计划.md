## Phase 1：数据层（四张表 + Room）

**目标**：完整的数据库层，可以编译通过

### 1.1 Entity 定义

按照文档第三章四张表逐一创建 Room Entity：

| 文件 | 内容 |
|------|------|
| `RecipeEntity.kt` | 方案表，字段对齐3.1 |
| `RecipeStepEntity.kt` | 方案步骤表，字段对齐3.2，含 `recipe_id` 外键 |
| `BeanEntity.kt` | 豆仓表，字段对齐3.3，`price_per_gram` 用计算属性（不存DB列） |
| `BrewLogEntity.kt` | 冲煮记录表，字段对齐3.4 |

**关键点**：
- 所有 ID 用 UUID 字符串
- 枚举字段用 `String` 存储，Mapper 做转换
- `price_per_gram` = `price ÷ net_weight`，不存列，Entity 中用 `@Ignore` 的计算属性
- `rating_tag` 不存列，Entity 中用计算属性根据 `rating` 映射

### 1.2 枚举类型定义

| 枚举文件 | 值 |
|----------|-----|
| `GrindSize.kt` | 粗 / 中粗 / 中 / 中细 / 细 |
| `StepActionType.kt` | 闷蒸 / 注水 / 搅拌 / 等待 |
| `Difficulty.kt` | 简单 / 中等 / 困难 |
| `BeanType.kt` | 单品 / 拼配 |
| `RoastLevel.kt` | 浅烘 / 中烘 / 深烘 |
| `BeanStatus.kt` | 未开封 / 已开封 / 已用完 |
| `Mood.kt` | 开心 / 平静 / 期待 / 疲倦 / 其他 |
| `Device.kt` | V60 / Origami / Bonmac / Kalita Wave / 其他（预设+可自定义，存String） |

> 注：器具(Device) 设计为预设枚举 + 自由文本字段，方便扩展。

### 1.3 DAO（数据访问对象）

| DAO | 主要查询 |
|-----|---------|
| `RecipeDao.kt` | CRUD + 按名称搜索 + 获取全部（按更新时间倒序） |
| `RecipeStepDao.kt` | 按 recipe_id 查所有步骤（按 order 排序）+ 批量插入/更新/删除 |
| `BeanDao.kt` | CRUD + 按状态筛选 + 获取全部（按烘焙日期倒序）+ 库存扣减更新 |
| `BrewLogDao.kt` | CRUD + 按时间倒序全部 + 按 bean_id 筛选 + 获取历史磨豆机列表 + 获取历史器具列表 |

### 1.4 数据库与类型转换器

| 文件 | 内容 |
|------|------|
| `AppDatabase.kt` | Room Database 类，版本=1，声明四表 |
| `Converters.kt` | 类型转换器（`List<String>` ↔ JSON 字符串 用于 flavor_tags） |
| `AppModule.kt` | 手动依赖注入（Application 中创建 Database 实例，各 ViewModel 通过 Application 获取 DAO） |

**验证标准**：编译通过，Room 在编译期生成 `*_Impl` 类。

### 2.1 方案列表页

- 保存后返回列表

### 2.3 方案详情页

## Phase 3：豆仓（P1）

**目标**：完整的豆子库存管理 CRUD + 库存扣减接口

### 页面清单

| 页面/Screen | 路由 | 功能 |
|-------------|------|------|
| 豆仓列表 | `beans/list` | 顶部汇总 + 豆子卡片列表 + FAB新建 |
| 新建/编辑豆子 | `beans/edit?beanId={id}` | 完整表单 |
| 豆子详情 | `beans/detail/{beanId}` | 所有字段展示 + 编辑/删除 |

### 3.1 豆仓列表页

**顶部汇总卡片**：
- 三列数据：总剩余重量（g） / 总剩余价格（元） / 已使用价格（元）
- 计算方式：
  - 总剩余重量 = Σ 所有豆子 current_weight（只算 status ≠ 已用完的）
  - 总剩余价格 = Σ (current_weight × price_per_gram)
  - 已使用价格 = Σ total_used_price

**豆子卡片**：
- 显示：名称 + 烘焙程度标签 + 风味标签（前2-3个） + 产地 · 豆种 + 当前剩余重量 + 克单价 + 状态标签
- 状态标签颜色：未开封(蓝) / 已开封(绿) / 已用完(灰)
- 卡片点击 → 详情页
- 卡片长按 → 删除确认

### 3.2 豆子编辑页

- 所有字段按文档3.3表
- 赏味期：预设快捷按钮（7/14/21/30天）+ 可自定义输入
- 风味标签：预设多选（水果/花香/坚果/巧克力/焦糖/香料/发酵/草本/茶感/酒感）+ 自定义输入补充
- 烘豆商：预设下拉（常见品牌）+ 可自定义
- 新建时 `current_weight` 默认 = `net_weight`，`status` 默认 = "未开封"
- `price_per_gram` = price ÷ net_weight，页面上实时计算显示

### 3.3 豆子详情页

- 显示所有字段
- 状态自动变更：若 `current_weight ≤ 0` 自动显示/变更为"已用完"
- 顶部工具栏：编辑 / 删除
- 可手动编辑 `current_weight`（用于纠正库存）

### 3.4 Repository 层（库存扣减）

创建 `BeanRepository.kt`：
- `deductStock(beanId, usedWeight)` → 扣减库存 + 累计已使用价格 + 检查是否用完
- 这个方法在 Phase 4 冲煮记录保存时被调用

**验证标准**：可以新建豆子 → 列表看到 → 详情查看 → 编辑 → 删除。汇总数据正确计算。

## Phase 4：冲煮记录（P1）

**目标**：完整的记录 CRUD + 豆子关联扣减 + 从引导跳转预填

### 页面清单

| 页面/Screen | 路由 | 功能 |
|-------------|------|------|
| 记录列表 | `brewlogs/list` | 按时间倒序卡片列表 + FAB新建 |
| 新建/编辑记录 | `brewlogs/edit?brewLogId={id}&recipeId={rid}&prefillData={json}` | 表单（支持预填） |
| 记录详情 | `brewlogs/detail/{brewLogId}` | 全部字段 + 另存为方案按钮 |

### 4.1 记录列表页

- 每张卡片显示：日期 + 豆子名 + 方案名（如有，灰色小字）+ 总耗时（格式 X分X秒）+ 星级评分
- 点击 → 详情页
- 长按 → 删除确认

### 4.2 新建记录（两种入口）

**入口一：手动新建**（从记录列表点 FAB）
- 空表单，所有字段由用户填写
- bean_id 和 bean_used_weight 必填

**入口二：从冲煮引导跳转**（Phase 5 完成后联调）
- 通过 Navigation 参数传入预填数据：
  - recipe_id（关联方案）
  - total_water（方案值）
  - water_temp（方案值）
  - grind_size（方案值）
  - device（方案值）
  - total_duration（实际计时）
  - ground_weight（方案粉重）
- 用户仍需手动填写：豆子、用豆量、实际注水量、磨豆机、地点、天气、评分、心情、感受、改进备注

### 4.3 记录表单关键逻辑

**豆子选择**：
- 下拉选择豆仓中的豆子（显示名称 + 剩余重量）
- 选择后自动显示该豆子的克单价

**自动补全（磨豆机/器具）**：
- 输入时查询历史记录中已填过的值
- 下拉建议列表
- 用户可选择快速填入或继续自由输入

**星级评分**：
- 可点击的五星评分组件
- 选择后自动生成 rating_tag（1→不太满意 ... 5→超级好喝）
- 实时显示生成的标签文字

**保存时触发扣减**（文档5.1）：
```
1. 根据 bean_id 查找对应豆子
2. current_weight -= bean_used_weight
3. total_used_price += bean_used_weight × price_per_gram
4. 若 current_weight ≤ 0，status = "已用完"
5. 保存豆子更新
6. 保存冲煮记录
```
全部在一个 `@Transaction` 中完成。

### 4.4 记录详情页

- 显示所有字段
- 星级 + 标签
- 底部按钮：【另存为方案】【删除】

**另存为方案**（文档5.3）：
- 若记录关联了方案 → 跳转新建方案页，预填步骤数据（复制关联方案的步骤），其他字段空
- 若未关联方案 → 弹出 Toast："该记录未关联方案，无法另存"

**删除记录**：
- 不回退豆仓库存（文档5.2）
- 弹出确认后删除

### 4.5 ViewModel

| ViewModel | 职责 |
|-----------|------|
| `BrewLogListViewModel` | 记录列表加载/删除 |
| `BrewLogEditViewModel` | 表单状态管理、豆子下拉加载、自动补全查询、保存+扣减 |
| `BrewLogDetailViewModel` | 记录详情加载、另存为逻辑、删除 |

**验证标准**：手动新建记录 → 选择豆子 → 保存 → 检查豆仓库存是否扣减 → 列表可见 → 详情查看 → 删除记录（库存不回退）。

## Phase 5：冲煮引导（P0 核心体验）

**目标**：从方案详情进入，经历准备→引导→完成，全程自动

### 页面清单

| 页面/Screen | 路由 | 功能 |
|-------------|------|------|
| 准备界面 | `brew/prepare/{recipeId}` | 提示语 + 开始按钮 |
| 引导界面 | `brew/guide/{recipeId}` | 圆环进度 + 步骤自动切换 + 语音 |
| 完成界面 | `brew/complete/{recipeId}` | 完成提示 + 去记录按钮 |

### 5.1 准备界面

- 显示方案名称
- 显示方案总步骤数 + 总耗时预估
- 提示语："准备好了吗？点击开始，全程自动引导，无需操作"
- 【开始】按钮 → 导航到引导界面

### 5.2 引导界面（核心）

**总计时**：
- 从点击【开始】起连续累加
- 格式 `MM:SS.s`（例：02:35.8）
- 用 `LaunchedEffect` + `while(true)` + `delay(100ms)` 驱动

**圆环进度**：
- Canvas 绘制圆弧，进度 = 当前步已用时间 / 当前步时长
- 每一步开始时圆环从0开始

**中心显示区**：
- 当前步骤动作类型（大号文字）：闷蒸 / 注水 / 搅拌 / 等待
- 当前步剩余时间（倒计时，秒）
- 总目标水量（固定，所有步骤水量加总）
- 阶段目标水量（当前步的目标水量，等待时隐藏）
- 平均流速（阶段水量 ÷ 阶段时长，单位 G/S）
- 步骤进度条：`第 X/Y 步`

**步骤自动切换**：
- 每步倒计时走完 → 自动切换到下一步
- 最后一步走完 → 导航到完成界面
- 全程无暂停按钮，不可回退
- 倒计时基于 `System.currentTimeMillis()` 差值计算，确保精准

**语音播报**：
- Android TTS (`TextToSpeech`)
- 每步开始：`"{动作类型}，注水{水量}克，持续{时长}秒"`
  - 例："注水，注水60克，持续15秒"
- 每步结束："叮"（播放短音频或 TTS 念"叮"）
- TTS 初始化在准备界面完成后预先加载

### 5.3 完成界面

- 大字："冲煮完成 🎉"
- 显示：总耗时（实际计时）、方案名
- 【去记录】按钮 → 跳转到新建记录页，携带预填数据（见4.2节入口二）
- 【返回方案库】文字按钮

### 5.4 ViewModel

| ViewModel | 职责 |
|-----------|------|
| `BrewGuideViewModel` | 管理引导状态：当前步骤索引、步骤倒计时、总计时、步骤列表加载 |

**关键状态**（StateFlow）：
```kotlin
data class BrewGuideState(
    val recipe: RecipeWithSteps,       // 方案+步骤
    val currentStepIndex: Int,         // 当前步骤索引 (0-based)
    val totalElapsedMs: Long,          // 总计时(毫秒)
    val stepElapsedMs: Long,           // 当前步已用时间(毫秒)
    val isRunning: Boolean,            // 是否运行中
    val isFinished: Boolean,           // 是否全部完成
)
```

**验证标准**：从方案详情 → 准备 → 开始 → 步骤自动切换 → 语音播报 → 完成 → 去记录，全流程跑通。

## 关键约束（防跑偏清单）

1. ✅ 方案不可被冲煮过程修改
2. ✅ 引导界面数据全部来自方案预设值
3. ✅ 步骤切换仅基于倒计时归零，无需用户点击
4. ✅ 语音播报固定格式
5. ✅ 记录与方案解耦，记录可不关联方案
6. ✅ 记录必须关联豆子，用豆量必填
7. ✅ 全部数据本地存储，无网络请求
8. ✅ 数字输入非负、水温≤100
9. ✅ 方案步骤支持拖动排序
10. ✅ 磨豆机/器具输入自动补全
