# 手冲咖啡APP —— AI编程执行计划书（最终完整版）


## 一、项目总览

| 属性 | 确认值 |
|------|--------|
| 运行方式 | 单机离线，不上架，不联网 |
| 数据存储 | 本地数据库（SQLite） |
| 底部导航 | 三个Tab：方案库 / 冲煮记录 / 豆仓 |
| 设计原则 | 方案是“菜谱”，记录是“实操日志”，两者解耦 |
| 核心逻辑 | 冲煮记录必须关联豆子，用豆量必填，保存时自动扣减豆仓库存 |


## 二、技术选型（已确认）

| 项目 | 选择 |
|------|--------|
| 语言 | Kotlin |
| 平台 | 安卓原生 |
| UI框架 | Jetpack Compose |
| 数据库 | Room（SQLite） |
| 状态管理 | ViewModel + StateFlow |

## 四、功能模块清单

### 模块一：方案库（默认首页Tab）

| 功能点 | 说明 |
|--------|------|
| 方案列表 | 卡片列表，显示：方案名称 + 总耗时（所有步骤时长相加）+ 粉水比；不显示步骤数 |
| 新建方案 | 进入编辑页，填写所有字段；步骤编辑器用四个积木（闷蒸/注水/搅拌/等待）组合，每个步骤填水量+时长；支持长按拖动调整顺序 |
| 编辑方案 | 点击方案进入详情页，可修改所有字段 |
| 删除方案 | 详情页或列表页提供删除按钮 |
| 方案详情页 | 显示完整参数 + 步骤列表；底部有【开始冲煮引导】按钮 |

### 模块二：冲煮引导（核心体验）

| 功能点 | 说明 |
|--------|------|
| 进入前准备 | 从方案详情页点击【开始冲煮引导】，进入准备界面，显示提示语，用户点击【开始】才启动 |
| 总计时 | 从点击【开始】起连续累加，格式 MM:SS.s（分:秒.十分之一秒） |
| 步骤自动切换 | 每步倒计时走完自动跳下一步，全程无需点击 |
| 界面布局 | 圆环进度 + 中心显示当前步骤信息 + 总目标水量（方案总和，固定） + 阶段目标水量（当前步，等待时隐藏） + 平均流速（按方案预设：阶段水量÷阶段时长，单位G/S） + 步骤进度条（显示当前第几步/共几步） |
| 语音播报 | 每一步开始时播报：“{动作类型}，注水{水量}克，持续{时长}秒”；每一步结束时“叮”一声 |
| 无法暂停 | 开始后不可暂停，不可回退 |
| 冲煮完成 | 最后一步走完后显示完成界面，展示“冲煮完成”和【去记录】按钮 |

### 模块三：冲煮记录（复盘日志）

| 功能点 | 说明 |
|--------|------|
| 记录列表 | 按时间倒序显示所有记录，显示：日期 + 豆子名 + 方案名（如有）+ 总耗时 |
| 新建记录（手动） | 点击【+】进入表单；所有字段按3.4表填写 |
| 新建记录（从引导跳转） | 冲煮完成后点击【去记录】，跳转到新建记录界面，自动预填：关联的方案、总注水量（方案值）、水温（方案值）、研磨设置（方案值）、器具（方案值）、总耗时（实际计时）、粉重（方案值）；其他字段（豆子、用豆量、注水量、磨豆机、地点、天气、评分、心情、感受、改进备注）需用户手动填写 |
| 自动补全（磨豆机/器具） | 输入时匹配历史记录中已填过的值，下拉选择可快速填入 |
| 记录详情 | 点记录进入详情页，查看所有字段 |
| 删除记录 | 记录详情页提供删除按钮（删除时**不**回退豆仓库存） |
| 另存为方案 | 记录详情页提供【另存为方案】按钮，将本次参数填入新建方案表单（步骤数据：如果关联了方案则复制方案步骤；如果没关联方案则无法另存，需提示“未关联方案，无法另存”） |

### 模块四：豆仓（咖啡豆库存）

| 功能点 | 说明 |
|--------|------|
| 顶部汇总 | 显示三个数字：总剩余重量（g）、总剩余价格（元）、已使用价格（元） |
| 豆子列表 | 每张卡片显示：名称 + 烘焙程度 + 风味标签（前2-3个）+ 产地 + 豆种 + 当前剩余重量（这包豆子的）+ 克单价 + 状态小标签（未开封/已开封/已用完） |
| 新建豆子 | 填写3.3表所有字段 |
| 编辑豆子 | 点卡片进入详情页，可修改所有字段（除price_per_gram自动重算外） |
| 删除豆子 | 详情页或列表页提供删除按钮 |
| 状态自动变更 | 当 current_weight ≤ 0 时，状态自动变为“已用完” |


## 五、关键业务逻辑（告诉AI“怎么联动”）

### 5.1 冲煮记录保存时的自动扣减逻辑

当一条冲煮记录保存时，系统自动执行：
1.根据 bean_id 查找对应的豆子

2.current_weight = current_weight - bean_used_weight

3.total_used_price = total_used_price + (bean_used_weight × price_per_gram)

4.如果 current_weight ≤ 0，status 自动变为“已用完”

5.更新豆子记录

### 5.2 记录删除时的处理

删除记录时，**不**回退豆仓库存（避免用户删除测试记录时污染库存数据）。如需修正库存，用户应在豆仓详情页手动编辑剩余重量。

### 5.3 另存为方案的逻辑

- 如果记录关联了方案：复制该方案的步骤数据到新方案中，其他字段（名称、器具、水温、粉重、研磨度、总注水量、难度、出处）由用户在新建方案表单中填写或修改。
- 如果记录未关联方案：提示“该记录未关联方案，无法另存”，按钮不可用或弹出提示。

### 5.4 星级与标签映射（固定）

| 星级 | 自动标签 |
|------|----------|
| ⭐⭐⭐⭐⭐ | 超级好喝 |
| ⭐⭐⭐⭐ | 非常好喝 |
| ⭐⭐⭐ | 好喝 |
| ⭐⭐ | 一般 |
| ⭐ | 不太满意 |


## 六、给AI的“绝对约束规则”（防跑偏条款）

将这些规则写在提示词的开头或结尾：

1. **方案不可被冲煮过程修改**。冲煮中的任何偏差不记录、不反馈、不修改方案。方案是静态菜谱。
2. **引导界面的每一步数据来源**：水量、时长、平均流速全部来自方案预设值，不做实时计算。
3. **所有步骤的切换逻辑**：基于倒计时归零自动触发，无需用户点击任何按钮（除了开始按钮）。
4. **语音播报内容固定格式**：“{动作类型}，注水{水量}克，持续{时长}秒”。
5. **记录与方案解耦**：记录可以关联方案（可选），也可以不关联。不关联也能保存。
6. **记录必须关联豆子**：冲煮记录中的 bean_id 和 bean_used_weight 均为必填，保存时自动扣减豆仓库存。
7. **所有数据本地存储**：无网络请求，无云同步。
8. **数字输入框基础校验**：不能为负数，水温不超过100℃。
9. **方案步骤支持拖动排序**：长按拖动重新调整顺序。
10. **磨豆机/器具输入框自动补全**：输入时匹配历史记录中已填过的值，下拉选择可快速填入。


## 七、页面间跳转关系
底部导航栏（三个Tab）：
Tab1: 方案库（默认）
Tab2: 冲煮记录
Tab3: 豆仓

方案库 → 点方案 → 方案详情页 → 点击【开始冲煮引导】→ 准备界面 → 点击【开始】→ 引导界面（全程自动）→ 完成界面 → 点击【去记录】→ 新建记录（预填）
方案库 → 点击【+】→ 新建方案

冲煮记录 → 点击【+】→ 新建记录（纯手动）
冲煮记录 → 点击记录 → 记录详情页 → 【另存为方案】→ 新建方案（预填步骤）

豆仓 → 点击【+】→ 新建豆子
豆仓 → 点击豆子 → 豆子详情页 → 编辑/删除


## 八、建议的AI输入顺序（分步喂给AI）

| 轮次 | 输入内容 | 目标 |
|------|----------|------|
| 第1轮 | 项目概述 + 技术选型 + 全部数据模型（四张表） | 建立数据库结构和Model层 |
| 第2轮 | 模块一（方案库）全部功能 | 完成方案的CRUD和详情页 |
| 第3轮 | 模块二（冲煮引导）全部细节 + 约束规则1-4条 | 完成引导核心流程 |
| 第4轮 | 模块三（冲煮记录）+ 联动逻辑（扣减库存+另存为）+ 自动补全 | 完成记录模块 |
| 第5轮 | 模块四（豆仓）+ 顶部汇总 + 状态自动变更 + 底部导航整合 | 完成所有模块联调和UI统一 |


## 九、功能优先级

| 优先级 | 模块 | 理由 |
|--------|------|------|
| P0 | 方案库（增删改查 + 步骤编辑） | 没有方案，引导无数据来源 |

| P2 | 豆仓顶部汇总（总剩余重量/总剩余价格/已使用价格） | 增强数据洞察 |
| P2 | 另存为方案 | 快速归档好配方 |
| P3 | 磨豆机/器具自动补全 | 提升录入效率 |
